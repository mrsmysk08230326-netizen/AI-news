# Claude Code 最新情報 - 2026年3月29日

## 今日のハイライト

2026年3月は Claude Code にとって歴史的な月となっています。**Auto Mode（自動許可モード）**のリサーチプレビュー公開、**ボイスモード**の全ユーザー展開、そして **Claude Cowork + Computer Use** が「Anthropic史上最大のローンチ」として注目を集めています。また、Claude Codeの年間換算収益が **25億ドル超** に達し、2026年初頭から2倍以上に急成長しています。

---

## 新機能・アップデート

### Auto Mode（自動許可モード）— リサーチプレビュー
- Anthropicが **Claude CodeのAuto Mode** をリサーチプレビューとして発表（3月12日前後に展開）
- 従来の「dangerously-skip-permissions」コマンドの拡張版で、Claudeがコーディング中のパーミッション許可を自動的に処理
- プロンプトインジェクション等の脅威はブロックしつつ、開発者が常時画面を見張る必要がなくなる
- 現在は **Claude Sonnet 4.6 および Opus 4.6** のみ対応
- **推奨**: サンドボックスや隔離コンテナ環境での使用
- 出典: [TechCrunch（2026年3月24日）](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/)

### Claude Cowork + Computer Use
- 3月23日、**Computer Use機能** がClaude CoworkとClaude Codeにリサーチプレビューとして公開（macOS限定）
- Claudeがアプリの起動、ブラウザ操作、スプレッドシートへの入力など、コンピューターを自律的に操作
- [Latent Space](https://x.com/latentspacepod/status/2037015564850520223) は「Anthropic史上最大のローンチ」と評価

### Code Review Tool（コードレビューツール）— 3月9日
- Anthropicが **Claude Code Code Review** をローンチ（[TechCrunch報道](https://techcrunch.com/2026/03/09/anthropic-launches-code-review-tool-to-check-flood-of-ai-generated-code/)）
- AIが生成したコードを自動解析するマルチエージェントシステム
- GitHubと統合し、プルリクエストを自動解析してコメントを直接コードに付与
- ロジックエラーや潜在的な問題をフラグアップ

### Voice Mode（ボイスモード）— 3月3日
- Claude Codeにネイティブのボイスモードが追加（[TechCrunch報道](https://techcrunch.com/2026/03/03/claude-code-rolls-out-a-voice-mode-capability/)）
- `/voice` コマンドで有効化、スペースバー長押しのプッシュトゥトーク方式
- 初期は約5%のユーザーに展開、以降順次拡大

### Analytics API
- Claude Code Analytics APIがリリース
- 組織が日次集計の使用メトリクス（生産性指標、ツール使用統計、コストデータ）にプログラムアクセス可能

### モデルアップデート
- Claude Opus 4 および 4.1 がモデルセレクターから削除、Opus 4.6 / Sonnet 4.6 に集約
- Sonnet 4.6 はコーディング性能が大幅向上（ユーザーテストで Sonnet 4.5 比70%が支持、Opus 4.5 比59%が支持）

## バグ修正・改善

### v2.1.87（最新）
- Cowork Dispatchでメッセージが配信されない不具合を修正

### v2.1.86（3月27日）
- APIリクエストに `X-Claude-Code-Session-Id` ヘッダーを追加（プロキシがセッション単位でリクエストを集約可能に）
- `.jj`（Jujutsu）と `.sl`（Sapling）をVCSディレクトリ除外リストに追加
- macOS キーチェーンキャッシュを5秒→30秒に延長（起動時のストール改善）
- `--resume` が「tool_use ids were found without tool_result blocks」で失敗する問題を修正
- 条件付きスキル/ルール設定時にプロジェクトルート外のファイルへのWrite/Edit/Readが失敗する問題を修正
- Windowsでスキル呼び出しのたびに不要な設定ディスク書き込みが発生する問題を修正
- 非常に長いセッションで `/feedback` 使用時にメモリ不足クラッシュが発生する可能性を修正
- `@` ファイルメンションのトークンオーバーヘッドを削減（生の文字列コンテンツのJSON二重エスケープを廃止）
- Bedrock、Vertex、Foundryユーザー向けのプロンプトキャッシュヒット率を改善

### 長期セッションの安定性
- PDFドキュメントを多数含む会話のコンパクションが失敗する問題を修正
- React CompilerのmemoCacheなど根本的なメモリ蓄積の原因を特定・解消
- 長時間セッションの複数のメモリリーク修正が集中的に実施

## 注目の使い方・Tips

### プラグインシステムの活用
Claude Codeのプラグインシステムが正式リリース済み。マーケットプレイスからカスタムコマンド、エージェント、フック、MCPサーバーを拡張インストール可能。

### 並行エージェント活用（Agent Teams）
1つのClaudeが「オーケストレーター」として複数のサブエージェントに指示を出しながらタスクを並列実行。複数の独立したClaudeインスタンスを別ウィンドウで並走させるスタイルが開発者コミュニティで人気。

### Remote Control（リモートコントロール）
2月25日公開のリサーチプレビュー。ローカルで動いているClaude Codeセッションに、スマートフォンやタブレット、別PCのブラウザからアクセスして作業継続が可能。

### CLAUDE.md の HTMLコメント非表示化
CLAUDE.md内のHTMLコメント（`<!-- ... -->`）は自動インジェクト時にClaude本体に表示されなくなった。ただし Read ツールで読み込んだ場合はコメントも可視。ドキュメント整理や運用メモの整理に活用できる。
