## Claude Code 便利機能・実用法

### 新機能・主要アップデート

**Parallel Sub-Agents（並行サブエージェント）**
チームリードセッションが複数の専門サブエージェントに作業を委譲し、それぞれが独立したコンテキストウィンドウで並行稼働するワークフロー。コンテキスト汚染を防ぎ、大規模タスクを高速化できる。まだ試していない人が最も多い機能とも言われている。

**Auto Memory（自動メモリ）**
Claudeが作業中に発見した知見・パターンを永続ディレクトリに自動記録する機能。`CLAUDE.md`がユーザーからClaudeへの指示書であるのに対し、Auto MemoryはClaudeが自分自身のために書くメモ。現在プログレッシブロールアウト中。

**Git Worktrees連携**
複数のClaudeインスタンスを同一プロジェクト内で同時稼働させる際にGit Worktreesを活用することで、コンフリクトを防ぎながら並行開発が可能。

### 実用Tips（X上で話題）

**Boris Cherny（Claude Code作者）から**（[@bcherny](https://x.com/bcherny/status/2007179832300581177)）
> 「正しい使い方は一つではない。好きにカスタマイズしてハックしてほしい」

**デバッグTips 10選**（[@aiedge_](https://x.com/aiedge_/status/2020226077592010902)）
- **Step Isolation**：全体を再生成せず、壊れたステップのみを再実行
- **Error Reproduction**：意図的に失敗を再現させて根本原因を把握
- **Rollback Prompts**：最後に成功したプロンプトまで戻り、変更を一つずつ再適用

**15のパワーTips**（[@alexchristou_](https://x.com/alexchristou_/status/1949870632524533939)）
- **デュアルウィンドウ**：Claude Codeを2窓同時起動（意外と知らない人が多い）
- **Plan Mode**（Shift+Tab）：実行前に計画を立てて「測ってから切る」
- **メッセージキュー**：Claude作業中に次の指示を先送りして連続タスクを効率化
- **画像ペースト**：`Ctrl+V`でスクリーンショットを直接送れる

**Longform Guide（[@affaanmustafa](https://x.com/affaanmustafa/status/2014040193557471352)）**
- `hookify` プラグインで会話形式からフックを作成可能（JSON手書き不要）
- `/fork` コマンドで会話を分岐させ、重複しないタスクを並行処理

### CLAUDE.md 設計のポイント
- **長すぎるCLAUDE.mdは逆効果**：重要ルールが埋もれてClaudeが無視するリスク。Claudeが指示なしで正しくやることは削除し、必須事項はフックへ。
- CLAUDE.mdの種類：リポジトリ共有 / 個人用（`Claude.local.md`）/ グローバル（`~/.Claude/`）/ ネスト型（サブディレクトリ）

### 公式ベストプラクティスガイド
Anthropicが「Claude Code: Best practices for agentic coding」ガイドを公開。数ヶ月の内部利用とコミュニティフィードバックを経たまとめ。（[@_catwu](https://x.com/_catwu/status/1913354716001739173)）

## 画像生成AI

### 主要モデル動向（2026年3月時点）

**GPT-4o / GPT Image 1.5（OpenAI）**
DALL-E 3から移行した新世代モデル。ChatGPT・SoraにPlus/Pro/Team/Free全ユーザーへの展開が完了。テキスト・ポートレート・アニメ・SF分野でトップ評価（[Artificial Analysis](https://x.com/ArtificialAnlys/status/1906713712381337732)）。GPT Image 1.5は4倍高速化、フォトリアリズムで業界トップ水準。APIでも顔・ロゴ・細部編集の高精度化が進む（[@OpenAIDevs](https://x.com/OpenAIDevs/status/1945538534884135132)）。

**Midjourney V7**
芸術的画像生成の王者として地位を堅持。ウェブ・モバイルアプリ対応に加え動画生成（最長21秒）も実装。Niji 7（2026年1月）でアニメ・イラスト特化機能を強化。

**FLUX.2 / FLUX.1.1 Pro（Black Forest Labs）**
Stability AI出身者が設立したBFLが開発。Meta提携で企業評価額32.5億ドル。FLUX.1.1 Proが生成時間4.5秒でリアリズム・商業用途でトップ評価。FLUX.2 DevはオープンウェイトモデルとしてDev向けに提供中。

**Stable Diffusion 3.5**
オープンソース志向で唯一無二の自由度。LoRA・ControlNet・カスタムモデルの深いカスタマイズが可能。8B Largeモデルから2.5B Medium（VRAM約10GB）まで3バリアント展開。

**Adobe Firefly 3**
ライセンス済みデータのみで学習した唯一のメジャーモデルで著作権侵害リスクがゼロ。商業プロジェクト向けに最適。Photoshop・IllustratorなどAdobe CCとの深い統合が強み。

**Grok Imagine API（xAI）**（[@xai](https://x.com/xai/status/2016751241226027302)）
2026年1月28日にAPIをローンチ。開発者向けエンドツーエンドのクリエイティブワークフローを提供。

## 動画生成AI

### 最大トピック：OpenAI Sora シャットダウン（3月25日）
Soraが3月25日に突然終了。コスト1日1,500万ドルに対し総収益210万ドル。Disneyが10億ドル投資契約を締結しMarvel・Pixar・Star Warsを含む200以上のキャラクターをライセンスしていたが、Sora終了とともにDisneyも撤退。後継モデル「Spud」が数週間以内に登場予定とOpenAIが発表。（[spectrumailab.com](https://spectrumailab.com/blog/veo-3-vs-sora-vs-runway-best-ai-video-generator-2026)）

### 主要モデル動向

**Google Veo 3.1**
Artificial Analysis Video Arenaで画像→動画・テキスト→動画の両カテゴリ1位を獲得（[ArtificialAnlys](https://x.com/ArtificialAnlys/status/1928318831761707224)）。ネイティブ同期音声生成、映画的リアリズム、高度なクリエイティブコントロールを実装。Sora 2・Runway Gen-4・Kling 3.0を押さえ複雑なマルチ要素プロンプトでのベンチマーク首位。利用可能：Google Cloud Vertex AI Studio / Flow / Gemini Advanced（Ultraプラン）。

**Runway Gen-4.5**
2026年1月リリース。従来モデルの弱点だったオブジェクトの一貫性問題を解決。「浮いた物理演算」問題を克服し、物体の重量・勢い・衝突がリアルに。クレジット制で映像品質ベンチマークをリード。さらに、RunwayはKling 3.0 / GPT-Image-1.5 / Sora 2 Proを含む全主要モデルをプラットフォーム内に統合（[@runwayml](https://x.com/runwayml/status/2024955482935525398)）。

**Kling 3.0（Kuaishou）**
2026年2月リリース。異なるカメラアングルをまたいだ被写体一貫性（3〜15秒のマルチショットシーケンス）が大きなブレイクスルー。8言語対応の音声リファレンス付きネイティブ音声生成。年収入$2.4億ペース（JPMorgan評価）。

**ByteDance Seedance 2.0**
2026年2月ローンチ。音声・動画の統合同時生成（後処理でなくネイティブ）を初実現。1プロンプトからのマルチショットストーリーテリング、8言語のフォネームレベルリップシンクを搭載。a16zも注目（[@a16z](https://x.com/a16z/status/2024533996928209126)）。

### 研究注目：動画AIは「見た目の品質」だけを最適化していた？
56研究者・32大学の共同研究が各動画AIモデルの推論能力を検証（[@heynavtoor](https://x.com/heynavtoor/status/2027980299804807655)）。スコア：人間97.4% / VBVR-Wan2.2 68.5% / Sora 2 54.6% / Veo 3.1 48.0% / Runway Gen-4 Turbo 40.3% / Kling 2.6 36.9%。「業界全体が間違った指標を最適化してきた」と指摘。
